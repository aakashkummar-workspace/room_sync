================================================================================
                        ROOMSYNC - COMPLETE FEATURE DOCUMENTATION
                           Smart Hostel / Roommate Automation App
================================================================================

App Name:       RoomSync
Platform:       Android (APK)
Framework:      Flutter 3.41.6 / Dart 3.11
Backend:        Supabase (PostgreSQL, Auth, Storage, Row Level Security)
State Mgmt:     Riverpod
Navigation:     GoRouter
APK Size:       ~53.9 MB

================================================================================
                          1. AUTHENTICATION & ONBOARDING
================================================================================

1.1  Email/Password Login
     - Users sign in with email and password via Supabase Auth
     - Error messages shown for invalid credentials or network issues

1.2  Sign Up
     - New account creation with name, email, password
     - Auto-creates a profile record in the "profiles" database table
     - User is set as "admin" role by default

1.3  Join Room Flow (3-Step)
     - Step 1: User enters room invite code
     - Step 2: Creates account with auto-generated password
     - Step 3: Joins the room and credentials are displayed in a modal
     - User can copy their credentials before proceeding

1.4  Session Persistence
     - App checks for existing Supabase session on launch
     - If session exists, auto-redirects to dashboard (no re-login needed)
     - Session survives app restarts

1.5  Auth Guard (Route Protection)
     - GoRouter redirect logic prevents unauthenticated access
     - All protected routes redirect to /login if not logged in
     - Login page redirects to / if already logged in

1.6  Error Recovery
     - Auth errors redirect to login instead of showing blank screen
     - Network errors shown with user-friendly messages

1.7  Logout
     - Clears Supabase session
     - Resets CurrentUser global state
     - Redirects to login screen

================================================================================
                           4. EXPENSES & SPLITS
================================================================================

4.1  Add Expense
     - Fields: Title, Amount, Category
     - Categories: General, Food, Utilities, Rent, Transport, Entertainment, Other
     - Select which members to split with (defaults to all)
     - Paid-by is automatically set to the current user

4.2  Auto-Split Calculation
     - Amount equally divided among selected members
     - Split amount = Total / Number of selected members
     - Each split recorded as a separate database entry

4.3  Split Tracking
     - Each expense shows list of splits
     - Split display: user name, amount owed, paid/unpaid status
     - Current user's own split auto-marked as paid

4.4  Settle Split
     - Mark individual splits as paid
     - Updates is_paid flag in expense_splits table

4.5  Delete Expense
     - Removes expense record and all associated splits
     - Cascading delete via database foreign keys

4.6  Financial Summary
     - My Spending: Total amount current user has paid for
     - Liability: How much current user owes to others (unpaid splits)
     - Receivable: How much others owe to current user (unpaid splits)
     - All calculated in parallel for performance

4.7  Category Colors
     - Each expense category has a distinct pastel color
     - Visual differentiation in expense list

4.8  Parallel Split Loading
     - All splits for all expenses fetched simultaneously via Future.wait
     - Previously: sequential loop (slow). Now: parallel (fast)

4.9  Notifications
     - When expense is added, all room members (except creator) receive notification
     - Notification format: "{name} added 'Title' - Rs.{amount}"
     - Notifications sent in parallel

4.10 Pull-to-Refresh
     - Swipe down to reload all expense data

16.2  Database Schema (14 Tables)
      1.  profiles           - User profiles (extends Supabase auth)
      2.  rooms              - Room/household info
      3.  expenses           - Shared expenses
      4.  expense_splits     - Individual expense splits
      5.  chores             - Household tasks
      6.  inventory          - Shared supplies/items
      7.  notes              - Sticky notes
      8.  messages           - Room group messages
      9.  direct_messages    - Private DMs between users
      10. house_rules        - Room rules
      11. notifications      - In-app notifications
      12. notices            - Announcements
      13. polls              - Voting polls
      14. poll_options       - Poll answer options

16.3  Security (Row Level Security)
      - All 14 tables have RLS enabled
      - Policies enforce:
        - Users can only read data from their own room
        - Users can only update their own profile
        - DMs only visible to sender and receiver
        - Notifications only visible to target user
        - Admins have additional write permissions

16.4  Performance Optimizations
      - Screens use CurrentUser.roomId instead of fetching full dashboard
      - Future.wait for parallel API calls (splits, polls, members)
      - Dashboard: 9 API calls run in parallel instead of sequential
      - Expense splits: all fetched in parallel (not one-by-one loop)
      - Poll options: all fetched in parallel
      - Notifications: sent in parallel to all members
